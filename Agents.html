Architecture

  • sceneMain.js orchestrates the public app: loads assets and layout, creates the character, mounts the portfolio overlay, and starts the shared game loop.
  • adminMain.js orchestrates the editor: auth gate, mutable state object, dirty-flag render loop, placement rules, and toolbar actions.
  • layoutStore.js is the persistence layer: load order is localStorage then published-layout.json then default-layout.json.
  • renderer.js is the canvas core: tiles first, then z-sorted furniture and character.
  • portfolio.js adds room-target camera transitions and modal content on top of the scene canvas.

Data Contracts

catalog.json defines 38 furniture types, 53 variants, and 4 categories: desks, chairs, wall, items.

default-layout.json is the blank-ish fallback. published-layout.json is the richer shipped room and currently contains 30 furniture placements.

Rules

  • Tile size is 16px.
  • Public scene uses fixed zoom 3.5; editor uses integer fit zoom.
  • All drawables are sorted by bottom-edge Y.
  • items render above the furniture beneath them.
  • mirror, centered, anchorBottom, and animation metadata are handled at render time.
  • Blocked occupancy is tracked as Set<"col,row">.
  • Furniture placement rules: desks on floor or wall-backed top edge, chairs on floor, wall items on wall, items unrestricted.
  • Character behavior is idle → random target → BFS path → walk → idle.

Tests

  • tests.html runs tileMap, character, layoutStore, and portfolio suites in-browser.
  • js/tests/run.js is the tiny harness with assert(), assertEqual(), and getResults().
  • githubPublish.test.js is drift: it imports missing js/githubPublish.js and is not loaded by tests.html.

Conventions

  • Use ES modules with explicit relative .js imports.
  • Functions are camelCase.
  • Furniture and variant IDs are UPPER_SNAKE_CASE.
  • DOM ids and CSS classes are kebab-case.
  • catalog.json is the runtime source of truth; per-folder manifest.json files are not consumed.
  • Local edit persistence key is pixel-office-layout.

Known Drift

AGENTS.md and CLAUDE.md can lag the code. docs/superpowers/ reflects design history, not guaranteed runtime behavior. Admin auth is only a client-side gate.
