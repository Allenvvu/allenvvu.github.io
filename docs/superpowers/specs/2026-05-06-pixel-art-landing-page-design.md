# Pixel Art Portfolio Landing Page — Design Spec

**Date:** 2026-05-06  
**Author:** Allen  
**Status:** Approved

---

## Overview

A personal portfolio landing page hosted on GitHub Pages (`github.io`). The entire page is a full-viewport HTML5 Canvas scene rendered in a 16-bit SNES RPG aesthetic — a cozy, top-down creative studio with ambient NPC animations. No user input required; the scene runs on its own. Tabs and portfolio sections will be added in future iterations; this spec covers only the landing page scene.

---

## Visual Design

- **Perspective:** Top-down orthographic, slightly elevated (classic SNES RPG 3/4 view)
- **Tile size:** 16×16 pixels drawn at 3× scale (48×48px on screen) via `imageSmoothingEnabled = false`
- **Room size:** 24×16 tiles (384×256 base px → 1152×768 at 3×), centered in the viewport
- **Floor:** Alternating dark/light wooden tiles drawn with Canvas `fillRect` — no image file needed
- **Walls:** Solid darker strip along the top edge of the room, drawn in code
- **Vignette:** Radial gradient overlay (dark edges, bright center) drawn over the full canvas
- **Name overlay:** CSS-positioned `<div>` over the canvas, bottom-center, showing name + title with semi-transparent dark background

**Color palette:**
| Element | Color |
|---|---|
| Floor tile A | `#7a4f2e` |
| Floor tile B | `#6b4226` |
| Wall | `#2a1f3d` |
| Wall accent | `#3d2f5a` |
| Page background | `#0d0d1a` |

## Architecture

```
personal-site/
├── index.html          ← canvas + name overlay + <script src="game.js">
├── game.js             ← all rendering and NPC logic (~250 lines)
└── sprites/
    ├── npc-1.png       ← blue shirt character (48×64)
    ├── npc-2.png       ← red hoodie character (48×64)
    ├── npc-3.png       ← teal shirt character (48×64)
    ├── bookshelf.png   ← wall bookshelf (16×32)
    ├── plant.png       ← potted plant (16×24)
    └── wall-art.png    ← framed painting (32×24)
```

Zero npm dependencies. Deploys directly to GitHub Pages as a static site.

## Sprite Specification

### NPC Character Sheets (3 files)

Each file: **48×64px RGBA PNG**, transparent background.

| Sheet region | Content |
|---|---|
| Row 0 (y 0–15) | Walk down — 3 frames |
| Row 1 (y 16–31) | Walk up — 3 frames |
| Row 2 (y 32–47) | Walk left — 3 frames |
| Row 3 (y 48–63) | Walk right — 3 frames |

Each frame: **16×16px**. Walk cycle plays frames `0 → 1 → 2 → 1` at 150ms per frame.

Placeholder sprites generated by `generate_sprites.py`. Replace any `sprites/npc-*.png` with custom art using the same sheet layout to update the scene.

### Furniture Sprites (3 files, static — no animation)

| File | Size | Placement |
|---|---|---|
| `bookshelf.png` | 16×32px | Left wall, 2 tiles from corner |
| `plant.png` | 16×24px | Bottom-right corner |
| `wall-art.png` | 32×24px | Center of top wall |

## NPC Behavior System

Three independent NPCs, each running the same state machine:

```
IDLE (0.5–2s random) → PICK_TARGET (random tile in room bounds) → WALK → IDLE
```

- **Walk speed:** 1 tile per 400ms (leisurely studio pace)
- **Room bounds:** NPCs stay within the inner tile area, avoiding the wall strip and furniture zones
- **Y-sort:** All NPCs are sorted by their Y position before each draw call so closer NPCs render on top of farther ones (depth illusion)
- **Frame animation:** Each NPC tracks its own `frameTimer` accumulator; advances every 150ms while walking, holds frame 0 while idle

---

## Rendering Pipeline (per frame)

1. Clear canvas
2. Draw floor tiles (code — alternating `fillRect`)
3. Draw wall strip (code — solid rect + baseboard line)
4. Draw furniture sprites (sorted back-to-front by Y)
5. Sort NPC array by Y position
6. Draw NPC sprites (current direction row, current frame column)
7. Draw vignette overlay (radial gradient)

The name/title overlay is a CSS `<div>`, not drawn on canvas — it sits above the canvas via `position: absolute`.

## index.html Structure

```html
<!DOCTYPE html>
<html>
<head>
  <meta charset="UTF-8">
  <title>Allen — Portfolio</title>
  <style>/* full-viewport canvas, overlay positioning */</style>
</head>
<body>
  <canvas id="game"></canvas>
  <div id="overlay"><!-- name + title text --></div>
  <script src="game.js"></script>
</body>
</html>
```

Canvas is sized to `window.innerWidth × window.innerHeight` and listens to `resize` to recompute the room offset (room is always centered).

## Future Extension Points

- Swap any `sprites/npc-*.png` with custom character art (same sheet format)
- Add more furniture by registering new sprite entries in `game.js`'s furniture array
- Add tabs/nav by overlaying HTML elements above the canvas (no changes to game.js needed)
- Promote to Phaser.js if collision maps, portals, or interactive objects are needed later
